Incorporating Feedback from Survey Participants

Previous dashboard: https://public.tableau.com/app/profile/thomas.lee6836/viz/SFCrime2018-Presentpost-pg-3-9_25_312_17431384901140/TimeSeriesPage?publish=yes

Summary

Visual and Structural Enhancements

  • Reduced Visual Clutter: Replaced complex stacked bar charts with clearer visual types (e.g., line charts, simplified bars), especially in the “Incidents by Neighborhood” view.
  • Split Visuals Across Pages: Spaced out visualizations into multiple pages to reduce user overwhelm.
  • Streamlined Labels and Titles: Eliminated vague labels like “miscellaneous” or “other” and revised titles for clarity and consistency.

Filter Functionality Improvements

  • Synchronized Filters: All filters now consistently update all relevant visuals.
  • Descriptive Tooltips: Dropdowns now include helpful explanations of their scope and functionality.
  • “Apply All” Button: Users can now reset or apply all filters in one click.
  • Removed Erroneous Options: Eliminated “null” crime types and invalid years like 2025 from filters.

Clarity and Accessibility Enhancements

  • Plain Language Descriptions: Simplified section descriptions for users with different levels of data literacy.
  • Expanded Tooltips and Data Definitions: Added hover explanations for crime types (e.g., property vs. person crimes) and clarified data ranking versus count distinctions.
  • Dynamic Hover Data: Tooltip values now reflect filter selections and show accurate, updated info.

Data Accuracy and Interpretability

  • Validated Chart Logic: Addressed arithmetic inconsistencies and ensured values align across visuals.
  • Automatic Sorting: Charts like “When Do Incidents Occur?” now default to descending order for clearer insight into crime frequency.

Demographic Information

Several participants suggested the inclusion of demographic, housing, mental health, or socioeconomic data. While these are valuable, they exceed the current project scope and we addressed this by clarifying that this data is intended to be supplementary.

Project Overview and Purpose

  • Simplified Language: The original text was rewritten to be more direct and welcoming. It avoids technical jargon and clearly communicates who the dashboard is for: community developers, policymakers, and funders.
  • Clarified Intent: The purpose now emphasizes equity, safety, and revitalization, aligning with the project's mission and stakeholder values.
  • Expectation Management: We clarify that the dashboard does not include (e.g., demographics, housing, income data) to help users understand its scope and encourage them to use it in conjunction with other tools.

Data Source and Timeframe

  • Specific Data Citation: Clearly notes the dashboard uses SFPD incident reports from 2018 to the present.
  • Definition Clarity: Mentions that the dashboard follows local classification systems, which adds legitimacy and helps orient users to how crimes are categorized.

How Can It Be Used

  • Rewritten for Accessibility: The bullet points were rewritten in plain language.
  • Specific User Tasks: They now list specific user tasks, such as identifying affected neighborhoods, analyzing trends by time, examining crime types and correlations.
  • Demographic Context Cue: While the dashboard doesn’t contain demographic data, the text now notes users should consider combining it with other datasets for broader decision-making, which was a recommendation from our Usability Report.

Key Findings Section

Visual and Structural Enhancements

This section was newly added and is a direct result of the usability feedback:
  • Navigation Buttons: The “View Crime Trends & Maps” and “Learn About the Researchers” buttons were added to guide users intuitively to next steps.
  • Layout Spacing and Hierarchy: Layout Spacing and Hierarchy: Clear separation between sections and consistent headings make the homepage easier to scan.

When Incidents Occur Improvements

Data Accuracy and Interpretability

  • Chart Validation and Synchronization: The “When Do Incidents Occur” section has been overhauled to ensure that the “Incident Classification by Year” and monthly incident line charts respond consistently to selected year filters. This resolves prior confusion where users expected synchronization across visual elements but encountered static or inconsistent charts instead.
  • Improved Tooltip and Hover Data: Users can now hover over bars and lines to view incident counts by category and month. These details dynamically reflect filter selections, which helps clarify discrepancies and improves real-time interpretability.
  • Automatic Sorting for Readability: The bar chart now automatically sorts crime types in descending order to help users quickly identify the most frequent incident types by year.

Temporal Patterns and Year Filter Functionality

  • Updated Descriptions in Plain Language: The section explaining the charts now uses layperson-friendly language to guide users. It clarifies the purpose of the dual chart layout—year-over-year incident totals above and month-by-month variation below—making the function of each visual more intuitive.
  • Clarified Year Filter Behavior: We addressed confusion around the "Year" dropdown by ensuring that the selected years apply clearly and consistently across all relevant charts. A note now prompts users to click "Apply" after selecting filters to ensure proper rendering.
  • Resolved Year Option Errors: Erroneous or misleading year entries (e.g., 2025 with incomplete data) were removed. Charts now only display valid years to prevent misinterpretation of trends.
  • Enhanced Layout Readability: Text crowding within the bar chart labels was addressed by adjusting label spacing, improving legibility, and reducing clutter, especially for crime classification categories like "Miscellaneous" or "Other".

These changes respond directly to user feedback that emphasized difficulty interpreting time trends and inconsistencies in filter behavior. The “When Do Incidents Occur” section now offers more reliable interaction and better supports community decision-making and temporal analysis.

Incidents Occurrence Heatmap Improvements

As part of our broader effort to improve temporal clarity and usability, we implemented the following changes:

Clarified Purpose and Interpretation

  • Descriptive Introduction Added: Added a descriptive introduction to help users understand the function of the heatmap.
  • Clarified Color Meaning: Clearly explained that darker red cells indicate higher crime counts, enabling users to visually identify peak periods of crime by type and month.
  • Contextual Use Explanation: Updated text to explain how this tool can support community planning and prevention initiatives.

Filter Functionality and Control Labeling

We improved data transparency by addressing user confusion about the information being displayed and controlled. We improved dropdown labeling for filters such as:

  • View Parameter (crime category vs. type)
  • Time Granularity (month/year)
  • Incident Year Filter (selection for valid years).

Tooltip Consistency and Data Values

  • Tooltips now show actual crime counts when hovering over any cell.
  • All values dynamically reflect filter selections, giving users accurate, contextual feedback in real time.

Accessibility and Visual Clarity

  • We improved color gradients for readability by all users, including those with visual impairments.
  • Reduced reliance on hovering alone by displaying crime counts directly in each cell based on a usability request from multiple participants.

Removal of Redundant or Unclear Classifications

  • Cleaned up labels such as “Miscellaneous” and ensured crime categories align with federal or local classification standards where possible.

Geographic Crime Distribution Improvements

This panel, which combines a heatmap of incident density by neighborhood with a bar chart of the top 15 locations, was refined in several key ways to improve interpretability, filter clarity, and comparative insights.

Improved Labeling and User Guidance

  • The section introduction was rewritten in plain language to help users quickly understand the dual-purpose of the visual: spatial intensity (via heatmap) and frequency ranking (via bar chart).
  • Additional guidance was included for applying filters and understanding performance expectations when large datasets (e.g., “All” years) are selected.

Synchronized Filter Controls

  • Dropdown filters for year, month, and weekday are now clearly labeled and update both the heatmap and bar chart simultaneously, aligning with improvements to other dashboard components.
  • A reminder to click “Apply” was added to ensure users finalize their filter selections, which was a frequent usability issue noted in feedback.

Data Clarity and Visual Contrast

  • Color scales for the geographic heatmap were adjusted for better contrast and to be more intuitive: darker areas now clearly indicate higher crime rates.
  • The color ramp includes a visible legend, helping users interpret quantitative values (ranging from 20 to 11,642) without guesswork.

Rankings Visualization

  • The "Top 15 Incident Locations" bar chart now includes:
    • Clearly labeled neighborhood names
    • Aligned numerical values for quick comparison
    • More even spacing and adjusted font for improved readability

Data Accuracy and Performance

  • Neighborhood boundary data was reviewed for alignment with city records to ensure that heatmap values are geographically accurate.
  • Performance improvements were made for faster rendering when filters span multiple years or city-wide views.

Crime Type Ranking Improvements

Clarified Narrative and Use Instructions

  • The section header and intro text were rewritten to clearly explain what the chart does not show exact counts, rather ranking order of crime types over time.
  • A tip was added to guide users that they should hover over the lines to view exact incident counts by year and crime type.

Differentiated Design for Visual Clarity

  • Color-coded lines were introduced and refined to visually separate each crime category across years.
  • Additional endpoint labels now display the most recent incident totals (e.g., "Larceny Theft: 23,028 incidents"). This makes it easier to interpret current trends without hovering.

Interpretation Aids for Non-Technical Users

  • Supporting text now includes examples (e.g., larceny consistently ranking high) to guide user interpretation of temporal rank shifts.
  • References were added to direct users to the “Incident Classification by Year” chart if they want total counts rather than ranks.

    • Performance and Interactivity
